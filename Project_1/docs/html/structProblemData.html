Detailed Description

Holds all parsed data ready to be consumed by the solver.

This is the output of the Parser and the input to the Graph builder. It owns all submissions, reviewers, and configuration.

Design decision — vectors vs. maps:
We store entities in vectors for cache-friendly iteration during graph construction, and additionally maintain unordered_maps for O(1) lookup by external ID (needed during parsing for duplicate detection and during output generation for reverse-mapping graph node indices → entity IDs).
