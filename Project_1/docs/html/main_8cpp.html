Detailed Description

Entry point for the Scientific Conference Organization Tool.

This file coordinates the full execution flow of the application and provides the user-facing interface of the project.

Two execution modes are supported:

  • Interactive mode (default): launches a terminal menu that allows the user to load input files, inspect parsed data, run the solver, and view results.
  • Batch mode (-b): runs the full pipeline non-interactively from the command line and writes the solution directly to the specified output file.

Interactive conventions used in this project:

  • Input test files are assumed to be located in: dataset/input/
  • Output files are written to: dataset/output/

  • With one argument, launches interactive mode and preloads that file.

◆ runInteractive()

int runInteractive ( const std::string &  preloadedFile = "")

Executes the program in interactive mode.

The interactive mode maintains two pieces of state:

  • the currently loaded problem data
  • the most recent solver result

If a file path is provided in preloadedFile, it is parsed before entering the menu loop.

Available actions include:

  • loading a file from dataset/input/
  • inspecting parsed submissions and reviewers
  • running the max-flow solver
  • running risk analysis
  • reprinting the last computed result
