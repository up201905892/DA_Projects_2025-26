Detailed Description

Builds the bipartite flow network from parsed problem data.

This class encapsulates the mapping between the problem domain (reviewers, submissions) and the graph node indices. It is the ONLY place where the node-index convention is defined:

*   Node 0           → Super-Source (S)
*   Node 1           → Super-Sink (T)
*   Nodes [2, m+1]   → Reviewers  (offset = 2)
*   Nodes [m+2, m+n+1] → Submissions (offset = m + 2)
*
Design decision: Direction S → Reviewers → Submissions → T
  • C(S, R_j) = MaxRev enforces the per-reviewer upper bound.
  • C(R_j, P_i) = 1 enforces binary assignment + domain matching.
  • C(P_i, T) = MinRev enforces per-submission demand.

Builds the complete flow network.

Parameters
dataParsed problem data (submissions, reviewers, config).
Returns
A Graph ready for Edmonds-Karp, with all edges and capacities set.
Edge construction summary:
  1. For each reviewer j: S → R_j with capacity MaxRev.
  2. For each (reviewer j, submission i) where domain matches: R_j → P_i with capacity 1.
  3. For each submission i: P_i → T with capacity MinRev.

Determines if a reviewer can review a submission based on domain matching.

The match depends on the current MatchMode (controlled by Config flags):

  • A reviewer's primary (and optionally secondary) expertise is checked against the submission's primary (and optionally secondary) domain.
  • A match exists if ANY active reviewer expertise equals ANY active submission domain.
Parameters
revThe reviewer.
subThe submission.
useRevSecConsider reviewer's secondary expertise.
useSubSecConsider submission's secondary domain.
Returns
true if at least one domain-expertise pair matches.
Complexity
— at most 4 comparisons (2 expertise × 2 domains).
