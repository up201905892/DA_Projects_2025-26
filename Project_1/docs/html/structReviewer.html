Detailed Description

Represents a reviewer (expert) available to review submissions.

Each reviewer has a unique identifier, a mandatory primary expertise domain, and an optional secondary expertise domain. These domains are matched against submission domains to build the bipartite compatibility graph.

Note
The reviewer does NOT store its assigned submissions — those are derived from the max-flow solution at extraction time.
Example (from Figure 1 of the project description):
Reviewer r;
r.id = 1;
r.primaryExpertise = 1;
r.secondaryExpertise = std::nullopt;
